會 SQL,

查詢交給 SQL,寫入交給框架。 Queries belong to SQL. Writes belong to the framework.

一個 SQL-first 的微型 ORM。用你已經會的 ANSI SQL 寫查詢,把繁瑣的 INSERT / UPDATE / DELETE 交給框架。幾乎不存在的學習曲線 —— 二十年來如一日。

The Philosophy

和 ORM 浪潮相反的取捨

2006 年,Hibernate 要求工程師先學會 HQL、Session 生命週期、mapping 儀式, 才能寫出第一行有生產力的程式。Modelet 反過來:尊重 SQL,只代勞真正繁瑣的部分。

SELECT 查詢交給 SQL

SELECT 本來就該用工程師已經會的 ANSI SQL 直接寫 —— 會下 subquery、join 就能開工。框架只把「一行資料 → 一個物件」搬回來,不發明新的查詢語言。

WRITE 寫入交給框架

真正機械、重複、容易出錯的是 INSERT / UPDATE / DELETE 的欄位羅列。 交給 Modelet:靠 entity 的 TxnMode 狀態機,自動產生語句、回填主鍵、蓋稽核欄位。

One Philosophy, Two Languages

同一份 entity,兩種語言逐行對照

Java 原生框架與 Python 移植共用同一套詞彙。宣告方式借自 Jakarta Persistence 註解, 讓兩邊讀起來一模一樣 —— 但只借詞彙,引擎始終 SQL-first。

Book.javaJava 17
@Table(name = "book")
public class Book extends AbstractEntity {

  @Column(name = "bookName")
  private String title;

  @Enumerated(EnumType.ORDINAL)
  private Grade grade;

  @Transient
  private Integer joinedRowCount;
}
book.pyPython 3.10+
@Table("book")
@dataclass
class Book(Entity):
    title: str | None = Column("bookName")

    grade: Grade | None = Enumerated(EnumType.ORDINAL)

    joined_row_count: int | None = Transient()
    # the FK IS the relationship — no @OneToMany

用起來也一樣:model.save(book) 做 INSERT 並回填 id,model.find(sql, params, Book) 用你寫的 SQL 撈回 entity。

What you get

小到一個下午讀得完,穩到二十年沒變

🗒️

SQL-first 查詢

subquery、join、group by 全都直接寫。框架不攔在你和資料庫之間。

🔁

TxnMode 狀態機

新建是 INSERT,存檔或查回自動轉 UPDATE,撥到 DELETE 就刪。一個 save() 全包。

🔑

主鍵自動回填

資料庫產生的 auto-increment id,insert 後自動塞回 entity。

📄

分頁容器

PagingElement 進、PageContainer 出,超過總頁數自動退回第一頁。

✍️

稽核欄位注入

AppEntity 自動蓋上 creator / createDate / modifier / modifyDate。

🧩

參數化到底

連 WHERE 的主鍵值都用 placeholder 綁定 —— 天生防 SQL injection。

☕

Java:Spring JDBC

站在 Spring JdbcTemplate 上,宣告式交易,Java 17 基線。

🐍

Python:零依賴

直接站在 DB-API 2.0 上,dataclass entity,contextvars 情境,約 600 行。

🏷️

熟悉的詞彙

@Table / @Id / @Column / @Transient / @Enumerated —— 借自 JPA,人人看得懂。
